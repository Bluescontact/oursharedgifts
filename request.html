Request

An ask, not a booking.

This page is not a calendar. There is no Book Now, no form with required fields, no funnel. The shape of the ask is yours; the infrastructure does the minimum to receive it cleanly.

You arrived asking about

If the ask feels rehearsed when you draft it, rewrite it. The page doesn't read for polish; it reads for what you actually need.

Three paths

For a gift offering

Pick one from the Offerings page or describe what you need that isn't on the menu. Send an email to the address below. What's useful to include is in the guidance section further down — not requirements, just structure that helps Kevin respond.

Send the ask →

For a consultation

Walk the Fit Filter first — five questions, no data captured, routes you to one of three outcomes. If the filter returns Fit, the next step (when the rail is live) is the calendar. If it returns Substrate first or Outside the tier, the filter tells you where to go instead.

Walk the filter →

For anything else

A question, a reference offering, a reading, a note. Just write directly.

Send a note →

What's useful to include

  • Who you are — name, and one line of context if it's relevant. Not a CV.
  • What you're asking for — the specific gift or question. If you're not sure, say that.
  • What kind of response would be useful — a yes/no, a referral, a longer conversation. Naming this saves both of you a round-trip.
  • Any constraint on timing — if there is one. If there isn't, leave it out.

What makes asks harder to respond to

  • Extensive backstory before the ask is named
  • Urgency framing ("could you respond by tomorrow") on a gift-tier request
  • Multiple asks layered together — easier if each is separate
  • Pre-emptive justification — Kevin reads the ask, not the case for why you deserve it

What happens after you send

Kevin reads the email. If it's a fit, you'll get a response with the next step — usually a time to talk, or a yes/no on a specific gift, or a referral. If it's not a fit, you'll get a clean decline that names why, often with a route to a different practitioner or resource. There is no follow-up sales sequence. There is no also you might be interested in. The exchange ends where it ends.

Capacity is real. See Current State for what's open right now. If capacity is closed when you send, the ask is held until capacity returns; the page updates when it does.
