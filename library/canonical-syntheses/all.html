Canonical Syntheses

Reference documents at the operational-substrate scale.

What follows is reference written for practitioners studying the framework to install it on their own work. The register is more technical than the rest of the library — it uses framework-internal vocabulary (pattern numbers, structural citations, cross-references) because the audience is practitioners who want to fork the substrate rather than receivers reading casually.

Receivers wanting surface-side reads should start with the index, the manuscript, or any of the framework texts in the main library.

Each synthesis below is a markdown file. Read it in your browser, or fork the repo to install the substrate on your own work.

Suggested reading order for pickup-practitioners

  1. Operating Principles Compilation — the constitutional layer (read this first)
  2. Substrate Cartography v2 — the inheritance map (what's where)
  3. Capacity Infrastructure Lens — the diagnostic apparatus
  4. Trust-Generation Operational Substrate — how trust generates without becoming a platform
  5. Receiver-Face Canonical Synthesis — what receivers experience at surfaces
  6. Discovery / First-Contact Substrate — the pre-arrival layer
  7. Pickup-Practitioner Onboarding Substrate — the phased operational sequence
  8. OSG v3 Recomposition Synthesis — worked example of applying the full substrate set to an existing site

The syntheses

  • Operating Principles Compilation
    Cycle 88 · constitutional substrate · ~3,000 words
    The eight operating principles in their canonical form, gathered from the agent spec into substrate-of-record. A structural finding surfaces in the compilation: the eight form a 4+4 tetrahedral pair — principles 1-4 organize the work; principles 5-8 enforce the disciplines. Constitutional vs. diagnostic distinction made explicit. Genesis trace for each principle. Pickup form: a pickup-practitioner writes their own eight at Phase 1.
  • Substrate Cartography v2
    Cycle 85 · inheritance map · ~5,000 words
    Updated substrate map covering cycles 50-84 (35-cycle interval since c49 v1). Maps the four arcs that produced the current substrate: post-crystallization · meta-tetrahedron recognition · capacity infrastructure lens · village market + Pattern Library v1.6. Names the operational-substrate-scale tier (didn't exist at c49). Current state of all top-level canonical syntheses, references, project clusters, briefs.
  • Capacity Infrastructure Lens
    Cycle 70 · diagnostic apparatus · ~6,000 words
    A way of seeing: distinguishes infrastructure that compounds substrate (capacity-yielding) from infrastructure that expands surface area (capacity-shaped extraction). The two are visually similar but structurally inverse. Six operational discriminators. Vertex-mapped + edge-named + face-failure analysis. Worked example applying the lens to a known platform that fails 0/6.
  • Trust-Generation Operational Substrate
    Cycle 84 · the gift form · ~6,500 words
    Eight trust-generation mechanisms gathered from across the framework: bio-as-substrate · explicit refusal enumerated · refused-pretense data disclosure · PFRC · clean exit on every surface · silent infrastructure · bounded scope · voice register continuity. Substrate-trust vs platform-trust as structurally inverse. Ten operational best practices. Anti-patterns mapped to shadow forms. Ten-step pickup form for other practitioners to install.
  • Receiver-Face Canonical Synthesis
    Cycle 83 · surface receiver-experience · ~5,500 words
    What the new surfaces ARE for receivers, in receiver-experience terms. Five receiver journeys mapped (discoverer · asker · claimer · submitter managing · pickup practitioner). Each surface's receiver-experience-name + what-it-is + discipline-for-translation-pass. Bio-as-substrate operationalized at receiver-face scale. Refused-vocabulary catalog for register audit.
  • Discovery / First-Contact Substrate
    Cycle 86 · pre-arrival layer · ~5,500 words
    The substrate before receivers arrive at any surface. Discovery in this framework operates BEFORE the receiver knows what they're discovering. Eight pathways · six trust-carrying mechanisms at the threshold · nine refused operations at propagation scale · three scales (global · local · travel-aware). Pickup form: a 10-step adoption sequence for practitioners installing discovery substrate.
  • Pickup-Practitioner Onboarding Substrate
    Cycle 87 · operational sequence · ~6,500 words
    The operational sequence in time that makes the framework actually receivable in another practitioner's hands. Six phases (Phase 0 self-discrimination · Phase 1 months 0-1 substrate-building · Phase 2 months 1-6 bio accumulation · Phase 3 months 6-12 first action-surface · Phase 4 ongoing cycle discipline · Phase 5 adjacency + propagation). Eight practitioner archetypes. Ten common drift moments with catch mechanisms. Portable vs. necessarily-yours distinction explicit.
  • OSG v3 Recomposition Synthesis
    Cycle 89 · worked example · ~5,000 words
    The first applied use of the complete surface-design substrate set at whole-site scale. Reads the existing oursharedgifts.org through everything learned and produces the canonical substrate-side rebuild. Names every existing surface's current alignment · new surfaces to add · what changes · what removes · what's preserved. Build sequence: substrate completion → translation pass → deploy → ongoing.

How to use these

If you are reading the framework for the first time, the suggested reading order above is the sequence that lets each synthesis rest on the prior one. Operating Principles first, then the inheritance map, then the lens, then the substrate set, then the worked example.

If you are forking this instance to hold your own radius, the technical handoff + the onboarding substrate together name the operational sequence.

If you are reading academically — feel free. The work is gift-form. Use what's useful. Adapt as needed. Credit sources.
