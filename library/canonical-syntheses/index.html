Threshold

A note before you continue.

What's behind this page is reference material more technical than the rest of this site. Eight documents · about forty thousand words across them.

They're written for practitioners studying the framework to install it on their own work — operating principles, the diagnostic lens, how trust generates without becoming a platform, how surfaces meet receivers, how discovery happens before arrival, the operational sequence for forking the substrate. The vocabulary is framework-internal: pattern numbers, structural citations, cross-references between cycles of substrate work. The audience is people who want to fork what's here rather than read the work as a finished thing.

If you came for the work itself — the manuscript, the framework texts, the practical pages — the main library and the index will serve you better. The work is on those pages; this is the scaffolding behind them.

A note on transparency.

This site does not gate the substrate behind paywalls, accounts, or commitment. The substrate is openly available; the click-through is here because the register changes through this door, and the change is worth naming.

Transparency is part of the architecture. The practitioner who writes the work decides what is exposed where — that's the operator-side of the design. This threshold is one such choice.
