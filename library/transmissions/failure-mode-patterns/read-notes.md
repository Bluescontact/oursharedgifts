# Read Notes — Twelve Failure-Mode Patterns

## Load-bearing distinction(s)

- **The break is not the problem; the missing vertex is.** A recurring failure is not a malfunction to be fought — it is the system reporting which of its four functional capacities has dropped out. The breakage is information, not the fault.
- **Surface fix vs. structural corrective.** Addressing the symptom deepens the trap (more documentation deepens the Transaction Trap; working harder accelerates the Burnout Cycle). The only corrective is to restore the dropped vertex.
- The hinge the whole piece turns on: **a failure pattern names which vertex is missing.** Twelve patterns, three per face, four faces — one per dropped vertex.

## Chosen geometry

**tetrahedron / minimum system** — with one vertex dropped (dimmed).

One line why: the argument *is* the four-vertex form failing one vertex at a time — the recognition pattern shows the complete tetrahedron with a single vertex gone dark and its edges collapsing, which is exactly the claim: failure = a missing vertex.

## Naming phrase

**WHEN A VERTEX DROPS** — subtitle: *the recurrence names which one.*

## Pull-quote candidates

1. A failure pattern names which vertex is missing. The corrective is to restore the vertex, not to fight the pattern.
2. The breakage is not the problem. It is the symptom of a missing vertex.
3. A Transaction Trap addressed with more documentation deepens the trap. A Burnout Cycle addressed by working harder accelerates the breakage.
4. The system is not fragile because of them. It is fragile *as* them.
5. A system that has failed the same way three times has named which vertex is structurally weak. The third recurrence is the diagnostic.
