# Connection × Technical

## Signature

Integrations technically work — the bytes flow, the schemas validate, the dashboards stay green. But you cannot point to a behavior that changes based on the data being passed. Logs accumulate without consumers. Events fire into queues no one is subscribed to. The form of exchange is intact; nothing is actually arriving inside it.

## Reading

Movement 2 + **doc2_natural_technicity.docx**. Technicity here means: a technology has Connection when its outputs become inputs to genuine response — not storage, not display, but actual change in the behavior of the receiving system. Without that, a technical artifact is a monument, not a living system.

## Practice

Map every integration in your system. For each, mark *exchanging meaning* or *passing bytes*. The "exchanging meaning" mark requires a specific answer to: *what changes in the receiver because of this data?* If the answer is "we have it on file," it's bytes.

## Intervention

Find one integration where systems are passing data but not exchanging meaning. Two moves, no third option. Either route the data to a real consumer that actually responds to it, or remove the integration. Do not leave it running as performance of contact. The half-life of dead integrations is decades, and they consume attention every time someone has to ask "what does this do?"

## Expected resistance

*"Someone might need it."* — If they did, they would already be using it. The hypothetical consumer is the same fiction as the hypothetical reader of unread documentation.

## Signal of success

Every integration in your system has a named consumer who would notice if it stopped. Removed integrations stay removed for at least one quarter without anyone asking for them back.
